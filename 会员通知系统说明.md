## 🔄 用户006自动递减机制

### 递减规则
用户006的到期时间会**每天自动递减**，形成循环：

```
第1天：T+3（还剩3天）→ 收到T-3天提醒
第2天：T+2（还剩2天）
第3天：T+1（还剩1天）→ 收到T-1天提醒
第4天：T+3（循环回到3天）→ 收到T-3天提醒
```

### 实现机制
1. **每日检测**：每次调用 `DB.init()` 时自动检测
2. **日期标记**：使用 `localStorage` 记录上次更新日期
3. **同日保护**：同一天内只更新一次
4. **自动循环**：到期后自动重置为3天
5. **数据库重置**：调用 `DB.reset()` 直接重置为T+3

### 技术细节
```javascript
// 每次页面加载时调用
DB.init() 
  → ensureDemoUser()
  → initSystemConfig()
  → updateUser006TrialTime() // 检查并更新用户006
```

## 🎯 功能特性

### 1. 自动检测机制
- 用户访问个人中心时，系统自动调用 `DB.checkAndCreateTrialReminders(userId)`
- 根据剩余天数自动生成相应的提醒消息
- 每种类型的提醒只会生成一次，避免重复通知

### 2. 消息通知界面
- **消息图标**：右上角铃铛图标
- **未读徽章**：红色圆点显示未读消息数量
- **消息列表**：点击铃铛查看所有消息
- **已读标记**：点击消息自动标记为已读
- **跳转功能**：点击消息跳转到会员开通页面

### 3. 消息状态
- **未读消息**：橙色背景 + 红点标记
- **已读消息**：灰色背景
- **全部已读**：一键标记所有消息为已读

## 🔄 数据恢复

### 重置数据库
调用 `DB.reset()` 会：
1. 清空所有数据
2. 重新初始化所有测试用户
3. 用户002：注册时间 = 当前时间，到期 = 当前+7天
4. 用户006：注册时间 = 当前-4天，到期 = 当前+3天

### 保持动态时间
- 用户002和006的时间都是动态计算的
- 每次重置数据库，时间都会更新为相对当前时间
- 确保测试环境始终有效

## 📝 注意事项

1. 消息通知基于localStorage存储，清除浏览器数据会丢失
2. 每种类型的提醒只会生成一次，避免重复
3. 时间计算基于客户端时间，确保系统时间准确
4. 测试时可使用 `DB.reset()` 重置所有数据

## 🆘 常见问题

**Q: 为什么看不到消息通知？**
A: 确保已登录用户006，并访问个人中心页面触发自动检测。

**Q: 如何清空所有消息？**
A: 访问测试页面，点击"清空所有消息"按钮。

**Q: 如何修改提醒时间点？**
A: 修改 `db.js` 中的 `checkAndCreateTrialReminders` 方法。

**Q: 如何添加新的消息类型？**
A: 使用 `DB.createNotification()` 方法创建自定义消息。
